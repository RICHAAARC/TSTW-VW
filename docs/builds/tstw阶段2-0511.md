## 一、文档定位

本文档是面向当前项目代码状态的阶段 2 完整修复契约。本文档可以独立使用，不依赖旧版“阶段 2”“阶段 2 真实测试”“阶段 2 修复”“项目结构调整”等文档。

本文档同时约束三类目标：

```text
1. 完成 real_video_vae_latent_probe 的真实 formal 测试链路；
2. 遵守 docs/file_organization.md 中定义的文件组织边界；
3. 强制将 Colab workflow 拆分为两个职责单一的 notebook，并使 notebook 与相关脚本命名满足 docs/naming_governance.md。
```

阶段 2 的最终目标不是“notebook 能运行完”，也不是“scaffold records 能生成”，而是完成如下真实闭环：

```text
真实 .mp4 source video
→ 真实 VAE encode
→ latent watermark embedding
→ 真实 VAE decode 为 .mp4
→ 真实 video attack，尤其是 ffmpeg H.264 / H.265 compression
→ attacked .mp4
→ 真实 VAE re-encode
→ tubelet / sync detection
→ fixed low-FPR threshold protocol
→ 真实视频质量指标与时序一致性指标
→ formal checker PASS
→ family result package 写回 Google Drive
```

阶段 2 完成后，才允许进入 trajectory_statistic_probe。阶段 2 未完成时，禁止通过 trajectory、DiT / Flow Matching 或 sampling-time 模块掩盖真实视频 VAE latent 闭环问题。
