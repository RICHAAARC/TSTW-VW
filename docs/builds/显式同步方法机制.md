# 显式同步方法机制：面向视频生成潜空间的时间同步 tubelet 水印方法

## 一、方法定位

本文提出一种面向视频生成潜空间的显式同步 tubelet 水印方法。该方法将视频水印的基本承载单元从单帧 latent 扩展为跨时间的 latent tubelet，并通过显式 temporal synchronization 恢复攻击后视频片段与原始水印时间轴之间的对齐关系。与逐帧水印相比，该方法重点解决视频水印中的时间失同步问题，即 temporal crop、local clip、frame dropping 和 speed change 等攻击会破坏检测端对水印时间索引的假设，使原本存在的水印信号无法被正确聚合。

本文的核心机制由四个部分组成：

```text
latent tubelet code
+ explicit temporal synchronization
+ aligned payload safety gate
+ fixed-FPR statistical detector
```

其中，latent tubelet code 负责将水印编码组织到视频潜空间的局部时空单元中；explicit temporal synchronization 负责恢复攻击后观测片段的时间参考系；aligned payload safety gate 负责判断候选对齐是否真正带来 payload evidence 增益；fixed-FPR statistical detector 负责在低假阳性率约束下给出最终水印判决。

该方法的关键设计原则是：同步分数只用于寻找候选对齐关系，不能直接作为水印存在的判决依据。最终 positive 结论必须来自对齐后的 payload evidence，并且必须通过 calibration negative 控制假阳性率。

---

## 二、问题定义

给定一个视频或视频生成模型输出的视频 latent 表示：

\[
z\in\mathbb{R}^{F\times C\times H\times W},
\]

其中，\(F\) 表示 latent frame 数，\(C\) 表示通道数，\(H,W\) 表示 latent 空间分辨率。对于真实视频实验，可通过视频变分自编码器（Variational Autoencoder，VAE）将视频 \(x\) 编码为 latent 表示：

\[
z=E_{\mathrm{VAE}}(x)。
\]

水印嵌入后得到 \(z'\)，再经 VAE 解码为水印视频：

\[
x'=D_{\mathrm{VAE}}(z')。
\]

攻击后视频记为 \(\hat{x}\)，检测端重新编码得到：

\[
\hat{z}=E_{\mathrm{VAE}}(\hat{x})。
\]

由于攻击可能改变视频的时间结构，检测端观测到的时间索引 \(\hat{\tau}\) 与原始水印时间索引 \(\tau\) 之间存在未知映射：

\[
\pi:\hat{\tau}\mapsto\tau。
\]

例如，temporal crop 对应未知时间偏移，local clip 对应只观察到局部时间片段，speed change 对应时间尺度变化，frame dropping 对应部分时间位置缺失。本文的目标是在未知 \(\pi\) 的情况下恢复可用时间对齐关系，并在固定假阳性率约束下判断视频是否包含密钥一致的水印。

因此，水印检测被定义为二元假设检验：

\[
H_0:y\sim P_{\mathrm{clean}},
\quad
H_1:y\sim P_{\mathrm{wm}}。
\]

其中，\(H_0\) 表示未水印视频，\(H_1\) 表示水印视频。检测器输出统计量 \(S_{\mathrm{final}}\)，并根据 calibration negative 上得到的阈值 \(\eta_\alpha\) 完成判决：

\[
S_{\mathrm{final}}(y)\geq \eta_\alpha
\Rightarrow
\text{watermarked},
\]

\[
S_{\mathrm{final}}(y)<\eta_\alpha
\Rightarrow
\text{clean}。
\]

## 三、核心思想

### （一）从逐帧水印转向 latent tubelet 水印

逐帧水印假设每一帧独立承载水印信号：

\[
c_f\rightarrow z_f。
\]

该假设在图像水印中较为自然，但在视频中存在结构性缺陷。视频攻击通常不只改变像素内容，还会改变时间索引关系。例如，攻击者删除前若干帧后，检测器看到的第1帧实际上可能对应原视频的第 \(k\) 帧；如果仍按原始第1帧的密钥位置读取水印，payload code 会发生系统错位。

为此，本文将视频 latent 划分为 tubelet 集合：

\[
\mathcal{G}=\{g=(\tau,i,j)\}。
\]

其中，\(\tau\) 表示时间 tubelet 索引，\(i,j\) 表示空间 tubelet 位置。每个 tubelet 对应局部张量：

\[
z_g\in\mathbb{R}^{L_t\times C\times P_h\times P_w}。
\]

其中，\(L_t\) 是时间长度，\(P_h,P_w\) 是空间 patch 尺寸。tubelet 将若干连续 latent frame 的局部空间区域作为一个时空单元，使水印天然具有跨帧冗余和局部片段可检测性。

### （二）从冗余重复转向显式时间同步

视频水印的关键难点不是简单增加重复次数，而是恢复攻击后的时间参考系。如果只增加 payload 冗余，而不解决检测端的时间对齐问题，裁剪后片段中的水印码仍可能被错误索引。

本文采用显式 temporal synchronization：检测器在候选时间变换集合 \(\Pi\) 中搜索最可信的时间对齐关系：

\[
\pi^\ast=\arg\max_{\pi\in\Pi}S_{\mathrm{sync}}(\pi)。
\]

在只考虑时间偏移与尺度变化时，该搜索可写为：

\[
(\Delta^\ast,\rho^\ast)
=
\arg\max_{\Delta,\rho}S_{\mathrm{sync}}(\Delta,\rho)。
\]

其中，\(\Delta\) 表示时间偏移，\(\rho\) 表示时间尺度。该机制使检测器能够在 temporal crop、speed change 和 local clip 条件下恢复候选时间参考系。

### （三）从同步峰值判决转向 payload 安全门控

同步搜索会扩大候选空间，也会在 negative 样本中产生偶然同步峰。如果直接将 sync score 作为 positive 判决依据，容易造成 attacked negative FPR 上升。因此，本文将 temporal synchronization 拆分为两个职责：

```text
alignment search:
  使用 sync candidate score 对 offset / scale 候选排序，寻找候选时间对齐关系。

payload safety gate:
  检查候选对齐是否真正带来 payload evidence 增益，并判断其是否允许影响 S_final。
```

也就是说，\(S_{\mathrm{sync}}\) 只回答“哪个时间对齐候选更可信”，而不直接回答“该视频是否有水印”。最终水印判决必须依赖 aligned payload evidence。

## 四、算法原语 1：Temporal-Synchronized Tubelet Code

### （一）Tubelet 划分

给定 latent video：

\[
z\in\mathbb{R}^{F\times C\times H\times W}，
\]

按照时间长度 \(L_t\) 与空间 patch 尺寸 \((P_h,P_w)\) 划分为 tubelet 集合：

\[
\mathcal{G}=\{g=(\tau,i,j)\}。
\]

每个 tubelet 对应局部张量：

\[
z_g\in\mathbb{R}^{L_t\times C\times P_h\times P_w}。
\]

当 \(L_t=1\) 时，该结构退化为 frame-level carrier；当 \(L_t>1\) 时，该结构成为跨帧 carrier。论文实验应将 \(L_t=1\) 作为 frame-level baseline，将 \(L_t>1\) 作为 tubelet 方法。

### （二）密钥方向生成

对每个 tubelet，使用密钥 \(K_{\mathrm{dir}}\) 生成单位投影方向：

\[
u_g=\mathrm{PRG}(K_{\mathrm{dir}},g),
\quad
\|u_g\|_2=1。
\]

检测时，tubelet 的投影响应为：

\[
a_g=\langle z_g,u_g\rangle。
\]

该方向由密钥和 tubelet index 共同确定，因此不同密钥下的投影方向应低相关，同一密钥下的嵌入与检测应保持一致。

### （三）Payload code

给定水印身份或消息 \(m\)，使用密钥 \(K_{\mathrm{pay}}\) 生成 payload 符号：

\[
c^{\mathrm{pay}}_g=\mathrm{PRC}(K_{\mathrm{pay}},m,g),
\quad
c^{\mathrm{pay}}_g\in\{-1,+1\}。
\]

payload code 用于承载身份信息或来源标识。对于单比特或来源认证任务，可以将 \(m\) 固定为密钥控制的身份标签；对于多比特消息任务，可以将 \(m\) 扩展为纠错编码后的 payload sequence。

### （四）Temporal synchronization code

对每个时间 tubelet 索引 \(\tau\)，生成同步符号：

\[
c^{\mathrm{sync}}_\tau=\mathrm{SyncCode}(K_{\mathrm{sync}},\tau),
\quad
c^{\mathrm{sync}}_\tau\in\{-1,+1\}。
\]

同步码应满足以下性质：

1. 自相关峰值清晰，便于 offset / scale search；
2. 不同密钥之间互相关低，降低误报；
3. 局部片段中仍保留足够同步结构；
4. 搜索空间扩大后仍可通过 calibration negative 控制 FPR。

候选同步码可以采用密钥生成的低互相关二元序列，也可以采用具有良好相关性质的序列族并进行密钥置乱。

### （五）Joint tubelet code

最终用于 tubelet 嵌入和检测的联合编码定义为：

\[
c_g=c^{\mathrm{pay}}_g\cdot c^{\mathrm{sync}}_\tau。
\]

该定义实现了 payload 与 synchronization 的耦合：payload 决定身份方向，sync 决定时间相位结构。检测时，只有在时间对齐关系正确时，payload code 才能被正确聚合。

### （六）Projection-margin embedding

嵌入阶段要求每个承载 tubelet 满足投影 margin：

\[
c_g\langle z_g,u_g\rangle\geq\alpha。
\]

若某个 tubelet 不满足该约束，则沿密钥方向做最小修正：

\[
z'_g=z_g+\left(\alpha-c_g\langle z_g,u_g\rangle\right)c_g u_g。
\]

该嵌入具有三点优势。第一，它与检测时的投影统计量严格对应，机制清楚；第二，它可以通过 \(\alpha\) 控制水印强度与失真之间的关系；第三，它便于在 synthetic latent 和 real VAE latent 中进行一致消融。

需要指出的是，\(\alpha\) 只能在 dev 或 calibration 阶段选择，不能基于 test positive 结果调参。

## 五、算法原语 2：Explicit Temporal Synchronization

### （一）攻击后的时间失同步

攻击后视频 latent 记为 \(\hat{z}\)。检测端无法直接知道观测索引 \(\hat{\tau}\) 对应原始水印时间索引 \(\tau\)。因此，需要搜索候选时间映射：

\[
\tau=\pi(\hat{\tau})。
\]

对于 temporal crop，可近似表示为：

\[
\pi(\hat{\tau})=\hat{\tau}+\Delta。
\]

对于 speed change，可扩展为：

\[
\pi(\hat{\tau})=\lfloor\rho\hat{\tau}+\Delta\rfloor。
\]

其中，\(\Delta\) 为时间偏移，\(\rho\) 为时间尺度。

### （二）同步响应计算

首先对每个观测时间位置 \(\hat{\tau}\) 聚合 tubelet 投影响应：

\[
\bar{q}_{\hat{\tau}}
=
\frac{1}{|\mathcal{G}_{\hat{\tau}}|}
\sum_{g\in\mathcal{G}_{\hat{\tau}}}
\langle\hat{z}_g,u_g\rangle。
\]

在候选 \((\Delta,ho)\) 下，同步响应为：

\[
S_{\mathrm{sync}}(\Delta,ho)
=
\sum_{\hat{\tau}}
\bar{q}_{\hat{\tau}}
\cdot
c^{\mathrm{sync}}_{\lfloor\rho\hat{\tau}+\Delta\rfloor}。
\]

检测端选择得分最高的候选：

\[
(\Delta^\ast,\rho^\ast)
=
\arg\max_{\Delta,\rho}S_{\mathrm{sync}}(\Delta,ho)。
\]

### （三）Coverage-aware candidate scoring

短片段或低覆盖候选可能由于少数 tubelet 偶然高响应而形成伪峰。为抑制这种情况，同步候选得分需要显式记录覆盖率和匹配数量：

\[
R_{\mathrm{cov}}(\Delta,ho)
=
\frac{N_{\mathrm{matched}}(\Delta,ho)}{N_{\mathrm{expected}}(\Delta,ho)}。
\]

可采用覆盖惩罚后的候选分数：

\[
S_{\mathrm{cand}}(\Delta,ho)
=
S_{\mathrm{sync}}(\Delta,ho)\cdot R_{\mathrm{cov}}(\Delta,ho)。
\]

或在候选筛选时要求：

\[
R_{\mathrm{cov}}(\Delta,ho)\geq r_0,
\quad
N_{\mathrm{matched}}(\Delta,ho)\geq n_0。
\]

实验记录中应保存：

```text
sync_alignment_coverage_ratio
sync_alignment_matched_count
sync_candidate_score_raw
sync_candidate_score_penalized
sync_estimated_offset
sync_estimated_scale
sync_search_space_size
```

这些字段用于解释同步候选，但不直接构成最终水印判决。

## 六、算法原语 3：Aligned Payload Safety Gate

### （一）候选对齐后的 payload score

在最佳候选 \(\pi^\ast\) 下，重新计算 payload evidence：

\[
S_{\mathrm{payload,aligned}}
=
\frac{1}{|\mathcal{G}_{\mathrm{obs}}|}
\sum_{g\in\mathcal{G}_{\mathrm{obs}}}
 c^{\mathrm{pay}}_{\pi^\ast(g)}
\langle\hat{z}_g,u_{\pi^\ast(g)}\rangle。
\]

未对齐或默认索引下的 payload evidence 记为：

\[
S_{\mathrm{payload,unaligned}}。
\]

候选对齐带来的 payload 增益定义为：

\[
G_{\mathrm{payload}}
=
S_{\mathrm{payload,aligned}}
-
S_{\mathrm{payload,unaligned}}。
\]

### （二）Safety gate 条件

候选对齐只有在同时满足以下条件时，才允许影响最终分数：

\[
S_{\mathrm{payload,aligned}}\geq\eta_{\mathrm{payload}},
\]

\[
G_{\mathrm{payload}}\geq g_0,
\]

\[
R_{\mathrm{cov}}\geq r_0,
\]

\[
N_{\mathrm{matched}}\geq n_0。
\]

形式化地，可定义：

\[
\mathrm{Gate}_{\mathrm{aligned}}
=
\mathbb{I}
[
S_{\mathrm{payload,aligned}}\geq\eta_{\mathrm{payload}}
\land
G_{\mathrm{payload}}\geq g_0
\land
R_{\mathrm{cov}}\geq r_0
\land
N_{\mathrm{matched}}\geq n_0
]。
\]

### （三）最终分数更新规则

为避免同步分数直接抬高 negative 样本，本文采用 payload-only rescue 规则：

\[
S_{\mathrm{final}}
=
S_{\mathrm{base}}
+
\mathrm{Gate}_{\mathrm{aligned}}\cdot
\max(0,G_{\mathrm{payload}})。
\]

其中，\(S_{\mathrm{base}}\) 是未应用 rescue 前的基础水印分数，可由 tubelet payload score 或 calibration-normalized tubelet score 给出。该规则保证：只有当候选对齐确实提升 payload evidence 时，才会增加最终水印分数；如果同步候选仅表现为高 sync score 而没有 payload 增益，则不会改变最终判决。

### （四）Negative safety

方法级 safety 需要在 calibration negative 上检查：

\[
\mathrm{negative\_rescue\_over\_threshold\_count}=0。
\]

同时报告：

```text
negative_rescue_over_threshold_rate
aligned_payload_negative_safety_status
aligned_payload_clean_negative_over_threshold_count
aligned_payload_attacked_negative_over_threshold_count
```

该审计用于证明 aligned payload safety gate 没有将 clean negative 或 attacked negative 推过 fixed-FPR 阈值。

## 七、算法原语 4：Fixed-FPR Statistical Detector

### （一）二元假设检验

检测问题定义为：

\[
H_0:y\sim P_{\mathrm{clean}},
\quad
H_1:y\sim P_{\mathrm{wm}}。
\]

检测器输出 \(S_{\mathrm{final}}\)，阈值 \(\eta_\alpha\) 由 calibration negative 上分位数确定：

\[
\eta_\alpha
=
\mathrm{Quantile}_{1-\alpha}
\left(
S_{\mathrm{final}}(\mathcal{D}_{\mathrm{calib-neg}})
\right)。
\]

其中：

\[
\mathcal{D}_{\mathrm{calib-neg}}
=
\mathcal{D}_{\mathrm{clean-neg}}
\cup
\mathcal{A}(\mathcal{D}_{\mathrm{clean-neg}})。
\]

即 calibration negative 必须同时包含 clean negative 与 attacked negative。这样可以吸收同步搜索带来的偶然峰值风险，避免只用 clean negative 低估实际 FPR。

### （二）Test-only evaluation

阈值确定后，在 test split 上固定使用，不允许再根据 test positive 或具体 attack type 调整阈值。需要报告：

```text
clean_negative_FPR
attacked_negative_FPR
clean_positive_TPR
attacked_positive_TPR
temporal_crop_TPR
local_clip_TPR
frame_dropping_TPR
speed_change_TPR
threshold_stability
confidence_interval
```

### （三）多 method variant 阈值

由于 `frame_prc`、`tubelet_only` 和 `tubelet_sync` 的 score distribution 不同，每个 method variant 可以具有独立 threshold，但必须共享同一 target FPR、同一 split、同一 attack matrix 和同一 calibration negative 规则。禁止 attack-specific threshold。

## 八、完整检测流程

### （一）嵌入流程

1. 输入视频 \(x\)，经 VAE 编码得到 \(z=E_{\mathrm{VAE}}(x)\)。
2. 按 \((L_t,P_h,P_w)\) 划分 latent tubelet。
3. 由密钥生成 \(u_g\)、\(c^{\mathrm{pay}}_g\) 与 \(c^{\mathrm{sync}}_\tau\)。
4. 构造联合编码 \(c_g=c^{\mathrm{pay}}_g c^{\mathrm{sync}}_\tau\)。
5. 对 selected carrier tubelet 执行 projection-margin embedding。
6. 得到水印 latent \(z'\)，并经 VAE 解码生成水印视频 \(x'=D_{\mathrm{VAE}}(z')\)。

### （二）攻击与重编码流程

1. 对 clean video 或 watermarked video 施加攻击 \(\mathcal{A}\)。
2. 得到攻击后视频 \(\hat{x}\)。
3. 检测端重新编码 \(\hat{z}=E_{\mathrm{VAE}}(\hat{x})\)。
4. 所有检测均基于 \(\hat{z}\)，不能直接使用未攻击 latent 替代。

### （三）检测流程

1. 在 \(\hat{z}\) 上计算基础 tubelet payload score。
2. 执行 explicit temporal synchronization，搜索候选 \((\Delta^\ast,\rho^\ast)\)。
3. 根据候选对齐关系计算 \(S_{\mathrm{payload,aligned}}\) 与 \(G_{\mathrm{payload}}\)。
4. 通过 aligned payload safety gate 判断是否允许 rescue。
5. 得到 \(S_{\mathrm{final}}\)。
6. 与 fixed-FPR 阈值 \(\eta_\alpha\) 比较，输出 detection decision。

## 九、方法边界与适用范围

本文方法适用于以下场景：

1. 视频或视频生成结果可以被映射到 latent space；
2. 检测端能够使用相同或兼容的 VAE encoder 进行重编码；
3. 攻击主要包括压缩、空间变换、时间裁剪、删帧、变速和局部片段检测；
4. 认证目标要求低误报率，而不是仅追求高召回率。

本文方法的核心主张限定为：

```text
显式 temporal synchronization 能够在 fixed-FPR 约束下提升 latent tubelet watermark 对时间失同步攻击和局部片段检测的鲁棒性。
```

因此，论文中不应将未进入正式 evidence 的轨迹统计、采样期约束或状态估计写成本文核心贡献。

## 十、实验设计

### （一）实验协议

所有实验采用 dev / calibration / test 三段式协议：

1. dev split 用于开发调试和候选配置初筛；
2. calibration split 仅用于阈值、标准化统计和预先声明的 gate 参数；
3. test split 仅用于最终评估。

样本角色包括：

```text
clean_negative
attacked_negative
watermarked_positive
attacked_positive
```

所有 method variant 必须共享同一 split、同一 attack matrix、同一 target FPR 和同一 table builder。

### （二）主要方法变体

必须比较以下变体：

```text
frame_prc
tubelet_only
tubelet_sync
tubelet_sync_aligned_payload_safety
```

其中：

1. `frame_prc` 表示逐帧 PRC baseline；
2. `tubelet_only` 启用 latent tubelet carrier，但不启用 temporal synchronization；
3. `tubelet_sync` 启用显式同步搜索；
4. `tubelet_sync_aligned_payload_safety` 启用显式同步搜索与 aligned payload safety gate。

### （三）攻击矩阵

实验应至少覆盖：

```text
no_attack
VAE_reconstruction
H.264 compression
H.265 compression
spatial_resize
crop_resize
Gaussian_noise
blur
temporal_crop
frame_dropping
speed_change
local_clip
```

其中，temporal crop、frame dropping、speed change 和 local clip 是验证显式同步机制的核心攻击；compression 与 spatial attack 用于验证水印在真实视频处理链路中的稳定性。

### （四）核心指标

鲁棒性指标包括：

```text
clean_negative_FPR
attacked_negative_FPR
clean_positive_TPR
attacked_positive_TPR
temporal_crop_TPR
local_clip_TPR
frame_dropping_TPR
speed_change_TPR
```

机制指标包括：

```text
sync_alignment_error
sync_peak_rank
sync_alignment_coverage_ratio
sync_alignment_matched_count
S_payload_aligned
S_payload_rescue_gain
negative_rescue_over_threshold_count
aligned_payload_negative_safety_status
```

质量指标包括：

```text
PSNR
SSIM
LPIPS
temporal_consistency_score
flicker_score
```

## 十四、支持的论文贡献表述

若实验成立，论文贡献可写为：

1. 本文提出一种 temporal-synchronized tubelet code，将视频水印承载单元从逐帧 latent 扩展为跨帧 latent tubelet，以缓解逐帧水印在时间攻击下的索引错位问题。

2. 本文提出显式 temporal synchronization 机制，通过 offset / scale candidate search 恢复攻击后视频片段的时间参考系，从而提升 temporal crop 和 local clip 条件下的水印检测鲁棒性。

3. 本文提出 aligned payload safety gate，将同步候选搜索与最终水印判决解耦，保证只有带来 payload evidence 增益且不破坏 negative safety 的对齐关系才能影响最终分数。

4. 本文在 fixed-FPR protocol 下进行统一检测评估，使用 clean negative 与 attacked negative 共同校准阈值，并在真实视频 VAE latent、时间攻击、空间攻击、压缩攻击和局部片段检测条件下验证方法有效性。

5. 本文通过系统消融证明 tubelet carrier、explicit synchronization 和 aligned payload safety 分别对应不同失效模式，并对最终鲁棒性具有独立贡献。

## 十五、失败条件与降级处理

若出现以下情况，不应将对应结论写入论文主张：

1. `tubelet_sync` 相比 `tubelet_only` 在 temporal crop 或 local clip 上无稳定增益；
2. aligned payload safety gate 导致 positive TPR 明显下降且无 FPR 改善；
3. attacked negative FPR 超过目标 FPR 约束；
4. quality metrics 显示视频质量明显塌缩；
5. temporal consistency 或 flicker score 显示明显时序闪烁；
6. 结果只能在 test split 调参后成立；
7. 表格不能从 records 和 thresholds 重建。

若某项攻击如 frame dropping 未能稳定提升，可在论文中将其标记为当前显式 offset / scale synchronization 的限制，并在 failure case analysis 中说明其非均匀时间映射特性。

## 十六、最终方法定义

本文方法可概括为：

> 一种面向视频生成潜空间的显式同步 tubelet 水印机制。该方法在 latent tubelet 中嵌入由 payload code 与 temporal sync code 共同控制的投影 margin 水印，通过显式 offset / scale synchronization 恢复攻击后片段的时间参考系，并利用 aligned payload safety gate 确保同步候选只有在产生真实 payload evidence 增益且不破坏 calibration negative safety 时才影响最终分数。最终检测在 fixed-FPR protocol 下完成，以保证水印认证的低误报可靠性。

简化公式为：

\[
c_g=c^{\mathrm{pay}}_g\cdot c^{\mathrm{sync}}_\tau,
\]

\[
(\Delta^\ast,\rho^\ast)=\arg\max_{\Delta,
\rho}S_{\mathrm{sync}}(\Delta,
\rho),
\]

\[
G_{\mathrm{payload}}=S_{\mathrm{payload,aligned}}-S_{\mathrm{payload,unaligned}},
\]

\[
S_{\mathrm{final}}=S_{\mathrm{base}}+
\mathrm{Gate}_{\mathrm{aligned}}\cdot\max(0,G_{\mathrm{payload}}),
\]

\[
\eta_\alpha=
\mathrm{Quantile}_{1-\alpha}
\left(S_{\mathrm{final}}(\mathcal{D}_{\mathrm{calib-neg}})
\right)。
\]

该定义完整覆盖了水印编码、显式同步、安全门控与 fixed-FPR 判决四个核心环节，可以直接作为论文方法机制章节的基础。
