## 一、阶段 1 的定位

阶段 1 应定义为 **synthetic video latent probe**，即“受控合成视频 latent 机制验证”。其核心目标不是接入真实视频生成模型，也不是验证真实视频质量，而是在阶段 0 已冻结的 `records`、`thresholds`、`tables`、`manifest`、`split` 与 `ablation` 协议基础上，首次引入具有明确算法含义的 temporal-synchronized tubelet code。

阶段 1 只回答一个核心机制问题：

```text
在 fixed low-FPR protocol 下，Tubelet+Sync 是否相对于 Frame-PRC 与 Tubelet-only 稳定提升 temporal crop、frame dropping、speed change 与 local clip detection 的鲁棒性，同时不造成 attacked negative FPR 失控。
```

因此，阶段 1 的最小结论应为：

```text
本阶段只证明：
synthetic video latent 可被稳定生成与复现；
frame_prc、tubelet_only、tubelet_sync 三类 variant 可在同一协议下运行；
tubelet projection score 与 sync score 不再是阶段 0 的 random score；
temporal attack matrix 可产生可审计的 attacked positive 与 attacked negative；
threshold 仍只由 calibration negative 产生；
local clip、temporal crop、tubelet length 与 sync alignment 的机制趋势可由 records 自动重建。
```

阶段 1 不应证明：

```text
真实视频 VAE latent 中机制成立；
Flow Matching trajectory statistic 具有独立增益；
DiT / Flow Matching sampling-time weak constraint 有效；
真实视频质量、语义一致性或运动一致性可控；
完整论文主表已经冻结。
```

阶段 1 的正确工程含义是：在不破坏阶段 0 协议层的前提下，将 `empty_watermark_method_placeholder` 与 `random_score_detector_random` 从正式验证路径中替换为可解释的 synthetic tubelet / sync 机制。

## 二、阶段 1 的命名总原则

阶段 1 仍不应使用 `_v1`、`_p1`、`stage1`、`probe1` 等弱语义编号。推荐沿用阶段 0 的稳定命名规则，但需要区分“正式 synthetic 机制字段”与“阶段 0 随机占位字段”。

### （一）正式阶段命名

推荐使用：

```text
construction_phase: synthetic_tubelet_sync_probe
protocol_name: fixed_low_fpr_calibrated_detection
method_family: temporal_tubelet_watermark
```

推荐 method variant 命名为：

```text
frame_prc
tubelet_only
tubelet_sync
```

阶段 1 的基础方法变体（primary method variant）只包括：

```text
frame_prc
tubelet_only
tubelet_sync
```

阶段 1 允许为了 `tubelet_length` sweep 生成派生消融变体（derived ablation variant），例如：

```text
tubelet_only_lt01
tubelet_only_lt02
tubelet_only_lt08
tubelet_only_lt16
```

这些变体不是新的主方法，必须在 records 中显式绑定：

```json
{
  "base_method_variant": "tubelet_only",
  "derived_variant": true,
  "ablation_axis": "tubelet_length",
  "tubelet_length": 8
}
```

主方法 records 必须写为：

```json
{
  "base_method_variant": "tubelet_sync",
  "derived_variant": false,
  "ablation_axis": null
}
```

不建议使用：

```text
frame_prc_v1
tubelet_sync_p1
stage1_tubelet
probe1_sync_detector
synthetic_probe_v1
```

如果历史 schema 中已有 `project_stage` 字段，可以保留兼容字段，但正式论文口径与配置口径应以 `construction_phase` 为准：

```json
{
  "project_stage": "stage_1",
  "construction_phase": "synthetic_tubelet_sync_probe"
}
```

其中，`project_stage` 仅作为历史兼容字段，不参与论文方法命名。

### （二）阶段 1 正式字段命名

阶段 1 中以下字段属于正式机制字段，不应带 `_random` 后缀：

```text
latent_shape
latent_generation_seed
latent_artifact_digest
tubelet_length
spatial_patch_size
embedding_margin
codebook_digest
sync_code_digest
sync_search_enabled
sync_estimated_offset
sync_ground_truth_offset
sync_alignment_error
sync_peak_rank
clip_length
```

原因是：阶段 1 的 `z\sim\mathcal{N}(0,I)` 是受控 synthetic probe 的正式实验对象，而不是阶段 0 的随机占位分数。其随机性由 seed、sample id、split 与 sample role 确定，并通过 digest 固定。

### （三）仍需保留后缀的字段

若阶段 1 仍保留阶段 0 的兼容入口，则必须继续显式标记：

```text
empty_watermark_method_placeholder
random_score_detector_random
identity_attack_placeholder
```

这些对象只能用于回归测试或兼容审计，不应进入阶段 1 的正式机制表格。

若某个字段在阶段 1 仍未真实实现，必须使用 `_placeholder` 后缀。例如：

```text
trajectory_observation_placeholder
quality_score_placeholder
motion_consistency_placeholder
semantic_consistency_placeholder
sampling_constraint_placeholder
```

阶段 1 中禁止将 `trajectory_score_random`、`quality_score_random` 等随机字段写入正式结论。trajectory、quality、semantic 与 motion 相关内容应保持禁用或占位，不得混入阶段 1 机制判断。

## 四、阶段 1 的构建顺序

### （一）第一步：新增 synthetic video latent backend

首先实现 `main/backends/synthetic_video_latent.py`，负责生成可复现的视频 latent：

\[
z\sim\mathcal{N}(0,I),\quad z\in\mathbb{R}^{F\times C\times H\times W}。
\]

该模块至少应支持以下配置项：

```yaml
latent_backend_name: synthetic_video_latent
construction_phase: synthetic_tubelet_sync_probe
latent_shape:
  frames: 32
  channels: 4
  height: 32
  width: 32
latent_distribution: standard_normal
latent_generation_seed: 20260507
latent_storage: npy_artifact
```

输出不应将完整 latent tensor 写入 records，而应写入 artifact，并在 records 中记录摘要：

```json
{
  "latent_backend_name": "synthetic_video_latent",
  "latent_shape": [32, 4, 32, 32],
  "latent_generation_seed": 20260507,
  "latent_artifact_relpath": "artifacts/latents/sample_test_watermarked_positive_000001.npy",
  "latent_artifact_digest": "sha256..."
}
```

要求：相同 `sample_id`、`split`、`sample_role`、`latent_generation_seed` 与配置 digest 必须得到相同 latent digest；不同 split 或 sample role 不得发生 sample id 泄漏。

---

### （二）第二步：实现 tubelet partition

新增 `main/methods/temporal_tubelet_watermark/tubelet_partition.py`，实现从 frame-level latent 到 tubelet-level latent 的划分。

每个 tubelet 记为：

\[
g=(\tau,i,j),
\]

对应局部张量：

\[
z_g\in\mathbb{R}^{L_t\times C\times P_h\times P_w}。
\]

配置项建议为：

```yaml
tubelet_partition:
  tubelet_length: 4
  spatial_patch_size:
    height: 4
    width: 4
  temporal_stride: 4
  spatial_stride:
    height: 4
    width: 4
  allow_partial_tubelet: false
```

要求：

```text
1. L_t=1 时必须可退化为 frame-level partition，用于 frame_prc；
2. L_t>1 时用于 tubelet_only 与 tubelet_sync；
3. 划分不得越界；
4. 同一配置下 tubelet index 必须稳定；
5. partition digest 必须进入 records 或 manifest。
```

---

### （三）第三步：实现 codebook

新增 `main/methods/temporal_tubelet_watermark/codebook.py`，负责生成三个确定性对象：

```text
PRG direction: u_g
payload code: c_pay_g
sync code: c_sync_tau
```

联合 tubelet code 定义为：

\[
c_g=c^{\mathrm{pay}}_g\cdot c^{\mathrm{sync}}_\tau。
\]

推荐配置项如下：

```yaml
codebook:
  direction_key: direction_key_synthetic_probe
  payload_key: payload_key_synthetic_probe
  sync_key: sync_key_synthetic_probe
  payload_bits: 64
  sync_code_family: keyed_binary_low_correlation
  direction_normalization: l2
```

输出必须可复现，并写入：

```json
{
  "codebook_digest": "sha256...",
  "sync_code_digest": "sha256...",
  "payload_digest": "sha256..."
}
```

要求：不得使用进程级全局随机状态；不得在 detect 阶段生成与 embed 阶段不一致的 codebook；不得将 payload code 与 sync code 混成不可解释的单一随机序列。

---

### （四）第四步：实现 projection-margin embedding

新增 `main/methods/temporal_tubelet_watermark/embedding.py`，实现阶段 1 的最小正式 embedding：

\[
c_g\langle z_g,u_g\rangle \ge \alpha。
\]

若不满足约束，则执行最小方向修正：

\[
z'_g=z_g+\left(\alpha-c_g\langle z_g,u_g\rangle\right)c_g u_g。
\]

推荐配置如下：

```yaml
embedding:
  embedding_rule: projection_margin
  embedding_margin: 0.25
  max_embedding_delta_norm: null
  record_embedding_diagnostics: true
```

必须记录：

```json
{
  "embedding_rule": "projection_margin",
  "embedding_margin": 0.25,
  "mean_projection_before": 0.013,
  "mean_projection_after": 0.251,
  "mean_embedding_delta_norm": 0.238
}
```

要求：

```text
1. clean_negative 不执行 embedding；
2. watermarked_positive 执行 embedding；
3. attacked_positive 必须由 watermarked_positive 经 attack 得到；
4. attacked_negative 必须由 clean_negative 经 attack 得到；
5. embedding 不得根据 test 结果调整 margin。
```

### （六）第六步：实现 temporal synchronization

新增 `main/methods/temporal_tubelet_watermark/synchronization.py`，先实现 offset search，再预留 scale search。

同步响应可定义为：

\[
S_{\mathrm{sync}}(\Delta)=\sum_{\hat{\tau}}\bar{q}_{\hat{\tau}}c^{\mathrm{sync}}_{\hat{\tau}+\Delta}。
\]

其中：

\[
\bar{q}_{\hat{\tau}}
=
\frac{1}{|\mathcal{G}_{\hat{\tau}}|}
\sum_{g\in\mathcal{G}_{\hat{\tau}}}
\langle \hat{z}_g,u_g\rangle。
\]

推荐配置：

```yaml
synchronization:
  enable_offset_search: true
  offset_search_min: -16
  offset_search_max: 16
  enable_scale_search: false
  scale_candidates: [1.0]
  sync_score_normalization: calibration_zscore
```

必须记录：

```json
{
  "sync_search_enabled": true,
  "sync_estimated_offset": -6,
  "sync_ground_truth_offset": -6,
  "sync_alignment_error": 0.0,
  "sync_peak_rank": 1,
  "sync_search_space_size": 33,
  "sync_search_space_digest": "sha256..."
}
```

要求：

```text
1. temporal crop 必须记录 ground-truth offset；
2. local clip 必须记录 clip_start 与 clip_length；
3. sync search 扩大搜索空间后，必须通过 calibration negative 控制 FPR；
4. 不允许只在 positive 上启用 sync search，而 negative 不启用；
5. sync search 参数不得根据 test attacked positive 调整。
```

### （七）第七步：实现 temporal attack matrix

新增 `main/attacks/temporal.py`，并在 `attack_registry.py` 中注册。阶段 1 最小攻击矩阵为：

```text
no_attack
temporal_crop
frame_dropping
speed_change
local_clip
latent_gaussian_noise
```

推荐配置如下：

```yaml
attack_matrix_name: temporal_attack_matrix
construction_phase: synthetic_tubelet_sync_probe
attacks:
  - attack_name: no_attack
    attack_params: {}
  - attack_name: temporal_crop
    attack_params:
      crop_start_candidates: [4, 8, 12]
      crop_length: 20
  - attack_name: frame_dropping
    attack_params:
      drop_rate: 0.25
      drop_policy: deterministic_keyed
  - attack_name: speed_change
    attack_params:
      speed_ratio: 1.25
      resample_policy: nearest
  - attack_name: local_clip
    attack_params:
      clip_lengths: [4, 8, 12, 16]
  - attack_name: latent_gaussian_noise
    attack_params:
      sigma: 0.05
```

每个 attacked event 必须记录 attack metadata：

```json
{
  "attack_name": "local_clip",
  "attack_params": {
    "clip_start": 8,
    "clip_length": 8,
    "original_frame_count": 32,
    "observed_frame_count": 8
  }
}
```

要求：attacked negative 与 attacked positive 使用同一 attack matrix；不得只攻击 positive，不攻击 negative；不得在不同 method variant 间改变 attack matrix。

通过标准为：


其中，第 5 至第 9 项属于机制通过标准；如果当前小规模实验暂时不稳定，则不得进入阶段 2，应先调整 tubelet partition、sync code、search range、embedding margin、calibration negative 规模与 attack matrix，而不是直接接入真实视频模型。

## 八、阶段 1 明确不做的内容

阶段 1 不应实现以下内容：

```text
1. 不实现真实视频读取、VAE encode 或 VAE decode；
2. 不实现 H.264 / H.265 compression；
3. 不实现 PSNR、SSIM、LPIPS、CLIP similarity 等真实视频质量表；
4. 不实现 Flow Matching trajectory statistic；
5. 不实现 trajectory-aware embedding；
6. 不接入 DiT / Flow Matching sampling hook；
7. 不声称 trajectory evidence 是核心贡献；
8. 不引入真实视频 watermark baseline comparison；
9. 不生成完整论文主表；
10. 不根据 test attacked positive 调整 threshold、margin、sync search range 或 fusion rule。
```

阶段 1 的边界必须保持清晰：它只验证 temporal-synchronized tubelet code 在受控 synthetic latent 条件下是否成立。

## 十、阶段 1 的最终判断口径

阶段 1 的最终判断不应写成“方法已经适用于真实视频生成水印”，而应写成：

```text
阶段 1 验证 temporal-synchronized tubelet code 在受控 synthetic video latent 条件下具备可解释机制趋势。具体而言，tubelet-level code 用于验证跨帧承载单元相对于 frame-wise code 的稳定性，temporal synchronization 用于验证未知时间偏移与局部片段检测条件下的对齐恢复能力，fixed low-FPR protocol 用于验证该机制不会通过牺牲 attacked negative FPR 获得虚假收益。
```

若阶段 1 通过，则可以进入真实视频 VAE latent probe。若阶段 1 不通过，则说明 temporal-synchronized tubelet code 的基础机制尚未成立，不应通过真实视频模型或更复杂的 Flow Matching 模块掩盖问题。
