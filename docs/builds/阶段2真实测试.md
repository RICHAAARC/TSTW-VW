# 阶段 2 真实测试构建说明：真实视频 + 真实 VAE + 真实压缩攻击 + 真实质量指标

## 一、任务定位

本文件用于指导 Codex 将当前阶段 2 的 `real_video_vae_latent_probe` scaffold 升级为可在 Google Colab 中执行的正式真实测试闭环。目标不是新增阶段 3 的 Flow Matching trajectory statistic，也不是重写阶段 1 的 tubelet / sync 方法，而是将当前“tensor-runtime scaffold”替换为真正的：

```text
真实 .mp4 视频
→ 真实视频读取与帧标准化
→ 真实 VAE encode
→ latent watermark embedding
→ 真实 VAE decode
→ 真实 .mp4 写出
→ ffmpeg H.264 / H.265 压缩与视频空间攻击
→ attacked video 重新 VAE encode
→ 阶段 1 tubelet / sync detector
→ fixed low-FPR threshold protocol
→ 真实视频质量指标与时序一致性指标
→ records / tables / report / failure gallery / tar.zst 归档
```

最终目标是使正式检查输出达到：

```json
{
  "status": true,
  "RealVideoVaeLatentDecision": "PASS",
  "BlockingReasons": [],
  "NextAllowedStage": "trajectory_statistic_probe"
}
```

当前项目状态应判定为：

```text
Stage2ScaffoldRunnable: PASS
Stage2FormalGate: PASS
Stage2RealVideoRuntime: FAIL
Stage2RealVAE: FAIL
Stage2RealCompressionAttack: FAIL
Stage2RealQualityMetrics: FAIL
Stage2FormalTestReady: FAIL
```

因此，本轮修复的核心不是补充更多 `.mp4` 文件，而是使代码真正使用 Google Drive 中已保存的数据集和模型，并在 formal 模式下禁止 placeholder 运行时被误判为阶段完成。

## 二、当前真实阻断点

### （一）`real_video_vae_latent.py` 仍依赖 synthetic backend

当前 `main/backends/real_video_vae_latent.py` 仍导入并实例化：

```python
from main.backends.synthetic_video_latent import SyntheticVideoLatentBackend
```

实际流程仍是：

```text
SyntheticVideoLatentBackend
→ synthetic latent
→ artifacts/videos/source/*.npy
→ artifacts/latents/encoded/*.npy
```

这不是阶段 2 formal 所需的真实视频流程。必须改为：

```text
dataset_manifest 中的 .mp4
→ video_io 读取
→ frame_preprocess 标准化
→ VAE encode
→ artifacts/latents/encoded/*.npy
```

### （二）当前 VAE backend 是 identity placeholder

当前 `main/vae/vae_registry.py` 只支持：

```text
video_vae_tensor_runtime
video_vae_backend_placeholder
```

当前 `VideoVAETensorRuntimeBackend.encode_video()` 和 `decode_video()` 均直接返回输入。这不能验证 VAE 编码噪声、VAE 解码失真，也不能支持 attacked video re-encode。

formal 模式必须新增并使用真实 VAE backend，例如：

```text
diffusers_autoencoder_kl_framewise
```

### （三）当前 video artifact 是 `.npy`，不是 `.mp4`

当前 `main/video/video_artifact.py` 使用：

```text
container = "npy"
codec = "tensor_npy"
```

formal 模式必须写出真实视频文件：

```text
artifacts/videos/source/*.mp4
artifacts/videos/decoded/*.mp4
artifacts/videos/watermarked/*.mp4
artifacts/videos/attacked/*.mp4
```

并记录：

```text
codec
container
fps
frame_count
height
width
pixel_format
file_sha256
```

### （四）当前压缩攻击是 tensor quantization placeholder

当前 `h264_compression` / `h265_compression` 只是对 tensor value 做量化，未调用 `ffmpeg`。formal 模式必须使用真实 `ffmpeg`：

```text
h264_compression → ffmpeg -c:v libx264 -crf ...
h265_compression → ffmpeg -c:v libx265 -crf ...
```

攻击必须作用于视频文件，而不是 latent tensor。

### （五）当前质量指标是 tensor proxy

当前 `quality_metrics.py` 使用 `.npy` tensor proxy 计算 PSNR / SSIM proxy，并禁用 LPIPS / CLIP。formal 模式至少必须实现：

```text
frame-based PSNR
frame-based SSIM
LPIPS
```

CLIP similarity 可以暂时 disabled，但 disabled 状态必须进入 manifest 和 report，不能作为完整质量结论。

### （六）当前 governance summary 固定 `INCONCLUSIVE`

当前 `main/analysis/real_video_vae_latent_artifacts.py` 中，非结构失败时仍固定写入：

```text
RealVideoVaeLatentDecision: INCONCLUSIVE
BlockingReasons: video_vae_backend_placeholder; real_video_runtime_not_enabled
quality_metrics_runtime: placeholder_tensor_video_metrics
temporal_metrics_runtime: placeholder_tensor_video_metrics
NextAllowedStage: remain_in_real_video_vae_latent_probe
```

formal runtime 完成后，必须由真实 records、manifest 与 metric runtime 判断生成 `PASS` 或 `FAIL`，不能继续固定为 `INCONCLUSIVE`。

实现：

```python
class DiffusersAutoencoderKLFramewiseBackend(VAEBackend):
    def encode_video(self, video_batch: np.ndarray, *, config: dict | None = None) -> LatentTensor:
        ...

    def decode_video(self, latent_batch: LatentTensor, *, config: dict | None = None) -> np.ndarray:
        ...

    def backend_metadata(self) -> dict[str, Any]:
        ...
```

输入输出规范：

```text
encode_video 输入：frames, H, W, 3；float32；[0, 1]；RGB
encode_video 输出：frames, C, H_lat, W_lat；float32；numpy 或 torch 后写入 .npy

decode_video 输入：frames, C, H_lat, W_lat
decode_video 输出：frames, H, W, 3；float32；[0, 1]；RGB
```

推荐实现逻辑：

```text
1. 从 local_model_root 加载 AutoencoderKL；
2. 将 RGB [0, 1] 转为 torch NCHW [-1, 1]；
3. 使用 AutoencoderKL.encode(...).latent_dist.mode() 或 mean 作为 deterministic encode；
4. 乘以 scaling_factor；
5. decode 时除以 scaling_factor；
6. decode 输出从 [-1, 1] 映射回 [0, 1]；
7. 按 batch_size_frames 分批处理；
8. formal 模式优先 cuda + float16；CPU 仅允许 smoke。
```

### （三）真实 `build_sample()` 流程

`build_sample(sample_id, split, sample_role)` 必须执行：

```text
1. 根据 split 与 sample_id 在 resolved dataset manifest 中找到 video_source_id；
2. 读取本地 .mp4；
3. 标准化 frame_count、fps、resolution；
4. 写入 source video artifact：artifacts/videos/source/<split>/<video_source_id>.mp4；
5. VAE encode 得到 encoded latent；
```

```text
working_latent_sample = source_sample
```
