## 十、P4：修复 runner decode / attack / re-encode 主闭环

### （一）目标

将 `main/protocol/real_video_vae_latent_runner.py` 从 tensor scaffold 主流程切换为 formal real video 主流程。

### （二）必须修改 1：formal runner 使用 real video attack registry

当前错误路径：

```python
attack_registry = build_real_video_attack_registry(attack_config)
```

必须改为：

```python
runtime_kind = "real_video" if run_mode == "formal" else "tensor_scaffold"
attack_registry = build_real_video_attack_registry(
    attack_config,
    runtime_kind=runtime_kind,
)
```

若 `runtime_profile` 或 runtime override 显式要求 real video，也应使用 `runtime_kind="real_video"`。

### （三）必须修改 2：decoded / watermarked video 写为 `.mp4`

formal 模式下，以下 artifact 必须为 `.mp4`：

```text
artifacts/videos/decoded/*.mp4
artifacts/videos/watermarked/*.mp4
```

禁止 formal 模式写：

```text
artifacts/videos/decoded/*.npy
artifacts/videos/watermarked/*.npy
```

`_cached_decoded_video_artifact()` 在 formal 模式下必须：

```text
latent
→ VAE decode
→ frames [T,H,W,3]
→ write_video_mp4()
→ probe_video_metadata()
→ file_sha256
```

不能再调用 `_write_video_tensor_artifact()`。

### （四）必须修改 3：attacked video 写为 `.mp4`

formal 模式下，以下 artifact 必须为 `.mp4`：

```text
artifacts/videos/attacked/*.mp4
```

如果 attack 是 H.264 / H.265：

```text
decoded_or_watermarked.mp4
→ ffmpeg attack
→ attacked.mp4
```

如果 attack 是 resize / crop-resize / blur / gaussian noise：

```text
decoded_or_watermarked.mp4
→ read_video_frames()
→ apply_frames()
→ write_video_mp4(attacked.mp4)
```

### （五）必须修改 4：decode / re-encode 不得强制 mock VAE

当前必须删除或条件化以下逻辑：

```python
runtime_profile = "smoke"
allow_mock_vae_backend = True
```

formal 模式下必须传入：

```python
runtime_profile = "formal"
allow_mock_vae_backend = False
```

并使用 runtime config 中的：

```text
vae_backend_name
diffusers_model_local_path 或 vae_model_id
```
